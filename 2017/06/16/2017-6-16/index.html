《机器学习》读书笔记

第一章 绪论

本章讲述的是机器学习的发展历程和基本思想，其中有以下有意义的观点。

名词：
奥卡姆剃刀：多种假设与观察相一致的时候，选择最简单的那个。
NFL定理（没有免费的午餐定理）：意味着无论算法多聪明，其总误差都与最笨拙的随机方法的误差相同。但是这个结论有一个条件——所有问题出现的机会相同／同等重要，算法选择是要根据问题的特性去改变的，具体问题具体分析。不存在可以解决一切问题的机器学习算法。
ILP：Inductive Logic Programming 归纳逻辑程序设计

人工智能的构成：

  1. 逻辑推理——“逻辑理论家”、“通用问题求解”等程序。
  2. 学习知识——有人来进行教学，专家系统。
  3. 深入学习——机器自己进行学习。

“从样例中学习”（“归纳学习”）：

  1. 决策树：模仿人的概念判定的树形流程，以信息论为基础，以信息熵的最小化为目标。（符号主义）
  2. 神经网络：BP是最主流的应用。最大的局限是试错性，参数的调节靠手动，但是却对结果有着极大的影响。 （连接主义）
  3. 支持向量机 (Support Vector Machine)(SVM)：（统计主义）。
  4. 核方法(kernel methods)：应用最广泛的机器学习内容。（统计主义）。
  5. 深度学习：狭义地说就是层数很多的神经网络，由于计算能力的增强和数据量的爆炸性增加才会兴起。（连接主义）。

第二章 模型评估与选择

经验误差与过拟合

过拟合：将训练样本本身的一些特点当作了所有潜在样本的普遍规律，假设变的太过复杂。
欠拟合：对于训练样本的一般性质尚未学好，假设太过简单。

解决方案：
欠拟合通过在决策树中扩展分枝、在神经网络中增加轮数等可以简单克服。
过拟合比较麻烦，不能避免，只能缓解。

评估方法

对于模型的评估方法需要使用泛化误差（在新样本上的误差）。因此使用一个测试集。以测试误差作为泛化误差的近似。

2017/6/29更新

留出法

即将数据集分成两半分别作为训练集和测试集。

需要注意的是，单次的留出法获得的结果不够准确，需要多次随机分组，重复评估，取得平均。

一般将2/3或4／5作为训练集，其余用来测试。

交叉验证

k折交叉验证，极限情况k=集合元素数m，称为留一法。

留一法与真实训练集D的训练集相近(只差一)，但是需要训练的模型太多了

自助法

用于见效样本规模的不同造成的影响，同时还可以高效地进行试验估计。

多用于初始数据量不够的情况。

调参与最终模型

调参要选择步长和范围。

使用整个训练集进行训练的出的模型才是最终模型。

性能度量

衡量模型泛化能力的评价标准。
要反映任务需求

回归任务最常用的性能度量是“均方误差”。

查准率、查全率与F1

用来度量挑出来的样品中有多少正确目标，或者是正确目标中有多少被挑了出来。这两个参数互相矛盾，一高一低。

根据学习器的预测结果对样例结果进行排序，从最可能到最不可能，然后按此顺序逐个把样例作为正例进行预测，就能得到一系列的查准(P)查全(R)率，以P为纵轴，R为横轴就能获得P-R曲线。

如果一个学习器的P-R图被另外一个完全包住，则可以断言后者优于前者。若发生交叉，则只能在具体PR下断言优劣，或者估算曲线包裹面积。

但是面积不好估算，所以使用平衡点(BEP)作为一个度量，就是P=R时的取值。

另外一个更常用的是F1度量。

其中，N为样例总数、TP为真正例(预测结果与实际结果均为正例)、TN为真反例(预测结果和实际结果均为反例)

F1还有一般形式，用于对查全率和查准率有偏好的情况

其中β>0，意味着查全率(R)对查准率(P)的相对重要性

有时会有多个二分类混淆矩阵，即进行了多次训练、测试，每次都会得到一个混淆矩阵；或者在多个数据集上进行了训练或者测试，希望估计其“全局”性能；或者是执行的是多分类任务，每个两两类别的组合都对应一个混淆矩阵等等，因此我们需要在n个二分类混淆矩阵上考察其PR

有两种做法：

一、 直接在各个混淆矩阵上分别计算出(P1,R1) (P2,R2)等，再计算平均值，就可以得到宏查准率(macro-P)、宏查全率(macro-R)、宏F1(macro-F1)

二、 先将各个混淆矩阵的对应元素进行平均，得到TP、FP、TN、FN的平均值，记为$\overline{TP} \overline{FP} \overline{TN} \overline{FN}$ ，再基于这一系列平均值计算出微查准率(micro-P)、微查全率(micro-R)、微F1(micro-F1)

ROC 与 AUC

在上节中提到过，我们可以将所有样例按照可能性排序，而分类器就是在其中选取某一点进行截断，前面的是正例，后面的是反例。

排序本身的质量好坏体现了学习器在不同任务下的“期望泛化性能”的好坏，ROC曲线就是从这个角度研究学习器泛化性能的工具。

ROC(Receiver Operating Characteristic)“受试者工作特征”，其横轴是“假正例率”(False Positive Rate 简称FPR)，其纵轴为“真正例率”(True Positive Rate 简称TPR)
